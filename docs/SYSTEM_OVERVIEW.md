# AgroHub Logistic — Описание системы

## Что это

Веб-платформа для торговли сельскохозяйственной продукцией с встроенной логистикой. Фермеры размещают товары, покупатели делают заказы, а система автоматически выстраивает оптимальные маршруты доставки и раздаёт их водителям. Всё в одном месте — маркетплейс, склад, диспетчерская и аналитика.

---

## Технологический стек

**Бэкенд** — Python 3.11 + FastAPI. Асинхронный, все запросы к базе через SQLAlchemy 2.0 async с драйвером asyncpg. Миграции — Alembic. Валидация данных — Pydantic v2. Фоновые задачи (уведомления, очистка) — APScheduler. PDF-накладные генерирует WeasyPrint через Jinja2-шаблоны. Файлы (фото товаров, подписи водителей) хранятся в MinIO через aiobotocore.

**Фронтенд** — React 18 + TypeScript 5, сборщик Vite. Серверное состояние — TanStack Query, глобальное — Zustand. Стили — Tailwind CSS. Карты — Leaflet + react-leaflet. Графики аналитики — Recharts. SPA с ленивой загрузкой маршрутов.

**База данных** — PostgreSQL 15.

**Инфраструктура** — Docker, деплой на Railway (бэкенд + фронтенд + Postgres + MinIO как отдельные сервисы).

**Авторизация** — JWT (access + refresh токены), 5 ролей.

## Роли пользователей

### Покупатель
Регистрируется, просматривает каталог товаров с фильтрацией по категориям и фермерам. Добавляет в корзину, оформляет заказ с указанием адреса доставки. Может отслеживать статус заказа — от «ожидает подтверждения» до «доставлен». После доставки оставляет отзыв на товар или фермера. Получает уведомления об изменении статуса.

### Фермер
Управляет своими товарами: создаёт карточки с фото, описанием, ценой, единицей измерения, минимальным заказом и остатком. Видит входящие заказы на свои товары, подтверждает или отклоняет их. Имеет доступ к базовой аналитике продаж — какие товары покупают, динамика выручки.

### Логист
Видит все подтверждённые заказы, готовые к отгрузке. Запускает алгоритм маршрутизации — система сама раскладывает заказы по машинам с учётом грузоподъёмности и объёма (FFD bin packing) и строит оптимальный порядок объезда точек (nearest-neighbor TSP). Логист просматривает результат на карте, при необходимости корректирует вручную и назначает рейсы водителям. Видит аналитику по всей платформе.

### Водитель
Видит свои назначенные рейсы в мобильном PWA-интерфейсе. На карте отображается маршрут с точками доставки по порядку. На каждой точке подтверждает доставку — ставит подпись пальцем на канвасе прямо в браузере. Подпись сохраняется как изображение. Если подписать некому — может отметить отказ или перенос.

### Администратор
Управляет пользователями — просматривает всех, меняет роли, блокирует аккаунты. Видит полную аналитику платформы: оборот, количество заказов, активность фермеров, загрузку транспорта. Имеет доступ ко всем разделам системы.

## Ключевые алгоритмы

**VRP-маршрутизация** — задача развозки товаров решается в два шага: сначала заказы распределяются по машинам алгоритмом FFD (First Fit Decreasing) с учётом веса и объёма груза, затем для каждой машины строится порядок объезда методом ближайшего соседа (nearest-neighbor TSP). Без сторонних солверов — всё на чистом Python.

**Статусная машина заказов** — `draft → pending → confirmed → ready → in_transit → delivered / cancelled`. Каждый переход доступен определённым ролям.
